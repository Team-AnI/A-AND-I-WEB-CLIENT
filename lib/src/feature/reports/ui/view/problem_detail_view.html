문제 설명

터미널 계산기는 터미널 입력을 통해서 연산된 결과값을 출력하는 프로그램입니다.

터미널 계산기는 실행이 되면 사용자에게 명령어를 입력받고 계산을 수행합니다. 명령어는 [COMMAND] [Value]의 구조를 갖고 있습니다. COMMAND는 각 연산을 의미하고, Value는 연산에 사용되는 값을 의미합니다. 명령어 종류에 따라서 Value는 존재하지 않을 수도 있습니다.

터미널 계산기는 아래와 같은 연산을 수행합니다. 그 외에 연산을 입력받는 경우에는 명령을 수행하지 않습니다.

  • ADD [Value]: 명령어는 Value를 더하는 연산입니다.
  • MINUS [Value]: 명령어는 Value를 빼는 연산입니다.
  • MUL [Value]: 명령어는 Value를 곱하는 연산입니다.
  • DIV [Value]: 명령어는 Value를 나누는 연산입니다.
  • CLOSE: 명령어는 프로그램을 종료하고 결과값을 출력하는 연산입니다.

터미널 계산기는 초기 값으로 0을 갖습니다.

CLOSE 명령을 통해 계산이 종료되면 터미널 계산기는 현재 값에 대해서 아래와 같이 출력하게 됩니다.

+ [Result], - [Result], [Result]

여기서, Result는 모든 연산이 끝난 결과값을 의미합니다. 결과값에 대해서 부호는 양수와 음수를 나타내며 0인 경우에는 부호를 출력하지 않습니다. 즉, 부호가 없는 경우는 0밖에 없습니다. 또한, 계산 결과는 모두 정수로 출력해야 합니다.

문제 요구 사항

  • fiber_manual_record 각 계산 절차를 함수를 통해서 분리해봅시다.
  • fiber_manual_record 프로그램의 시작은 main() 함수를 통해서 시작됩니다.
  • fiber_manual_record 사용자 명령어를 처리하는 input() 함수를 정의해봅시다.
  • fiber_manual_record 계산을 수행하는 calculate() 함수를 정의해봅시다.
  • fiber_manual_record 결과를 출력하는 display() 함수를 정의해봅시다.

학습 정리 목표

  • fiber_manual_record 함수의 정의와 매개변수 활용 능력 향상
  • fiber_manual_record 조건문과 반복문을 활용한 로직 설계 이해
  • fiber_manual_record 표준 입력(stdin)을 통한 문자열 파싱 이해
  • fiber_manual_record 출력 형식 규격에 따른 데이터 포맷팅 실습
